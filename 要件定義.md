# 業務時間集計アプリ 要件定義書

## 1. 概要
本アプリは、ブラウザ上で動作する業務時間計測ツールである。  
ユーザーが登録した「業務・作業内容」をボタンとして表示し、ボタン押下により業務時間の計測を開始・終了できる。  
計測データは SQLite に保存し、後で集計可能とする。

---

## 2. 対象ユーザー
- 個人利用者  
- 小規模チーム  
- 日々の業務時間を簡易に記録したいユーザー  

---

## 3. 利用環境
- Webブラウザ（Chrome / Edge / Firefox / Safari）
- サーバー側で SQLite を利用（ローカルまたは簡易サーバー）

---

# 4. 機能要件

## 4.1 業務・作業内容の登録・管理
- ユーザーは入力欄に業務名を入力し、「登録」ボタンで登録できる。
- 登録された業務は画面上にボタンとして縦に並んで表示される。
- **同名業務の登録は不可とし、エラー（アラート）を表示する。**
- **不要になった業務を画面上から非表示（アーカイブ）にする、または論理削除できる機能を提供する。**

---

## 4.2 計測開始
- 業務ボタン押下で計測を開始する。
- 計測開始時に以下を記録する：
  - 業務ID
  - 開始日時（システム日時）
- 計測中は他の業務ボタンが押されるまで継続する。

---

## 4.3 計測終了
- 初期状態では「終了」ボタンのみが存在する。
- 「終了」ボタンは現在計測中の業務を終了させるためだけに使用する。
- 終了時に以下を記録する：
  - 終了日時（システム日時）
  - 計測時間（終了日時 − 開始日時）
- 終了後は計測中状態が解除される。
- **日またぎの計測仕様について：**
  - **計測中に日付が変わった場合でも、基本は1つの連続したレコードとして扱う方針とする（必要に応じて日別の集計表示を行うフロントエンド側でまたぎ分を適宜分割して計算する）。**

---

## 4.4 計測切り替え
- 計測中に別の業務ボタンが押された場合：
  1. 現在の業務の計測を終了  
  2. 新たに押された業務の計測を開始  

---

## 4.5 履歴の確認と修正
- **当日の計測履歴・タイムラインを画面上で確認できる機能を提供する（今日どれだけ何の作業をしたかを一覧表示）。**
- **誤って計測した記録の削除や、開始・終了時刻を手入力で事後修正できる機能を提供する。**

---

## 4.6 計測データの保存（SQLite）
### テーブル：users（ユーザー管理）※小規模チーム対応を想定
| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | INTEGER PK | ユーザーID |
| name | TEXT | ユーザー名 |

### テーブル：tasks（業務マスタ）
| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | INTEGER PK | 業務ID |
| user_id | INTEGER | ユーザーID（FK）※個人利用に限定する場合は不要 |
| name | TEXT | 業務名 |
| is_deleted | INTEGER | 論理削除フラグ（0: 有効, 1: 削除済み） |

### テーブル：records（計測ログ）
| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | INTEGER PK | 記録ID |
| user_id | INTEGER | ユーザーID（FK）※個人利用に限定する場合は不要 |
| task_id | INTEGER | 業務ID（FK） |
| start_time | TEXT | 計測開始日時（システムのローカル時刻のISO8601形式：YYYY-MM-DD HH:MM:SSで保存） |
| end_time | TEXT | 計測終了日時（同上。計測中は NULL 許容） |
| duration_sec | INTEGER | 計測時間（秒。計測中は NULL 許容） |

---

# 5. UI要件

## 5.1 画面構成
- 上部：業務名入力欄 + 登録ボタン
- 中央：登録済み業務ボタン一覧（縦並び） ＋ 業務の編集/削除ボタン
- 下部：当日の計測履歴（タイムライン）表示
- 最下部：終了ボタン（常に表示）

---

## 5.2 業務ボタン
- ボタンは縦に並べて表示する。
- 計測中の業務ボタンは強調表示（色変更など）。
- ボタン押下で計測開始。

---

## 5.3 終了ボタン
- 常に画面最下部に固定表示。
- 計測中の業務がある場合のみ有効化。
- 押すと計測終了処理を実行。

---

# 6. 非機能要件

## 6.1 性能
- ボタン押下から計測開始・終了処理まで 1 秒以内に応答する。

## 6.2 信頼性
- ブラウザリロード時も計測状態が保持されるよう、状態管理を行う（SQLite 側の records テーブルの end_time が NULL のものを進行中ステータスとして扱う）。

## 6.3 保守性
- 業務追加・削除機能の拡張が容易な構造にする。

## 6.4 例外処理（エラーハンドリング）
- **サーバーとブラウザの通信が瞬断・エラーとなった場合は、画面上にエラーメッセージ（アラート等）を表示し、計測開始・終了処理を無効化することでデータの不整合を防ぐ。**
