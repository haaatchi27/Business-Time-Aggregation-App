## インストール方法

### 方法1: Docker（推奨）

Docker がインストールされていれば、すぐに使い始められます。

```bash
# 1. リポジトリをクローン
git clone https://github.com/haaatchi27/Business-Time-Aggregation-App
cd Business-Time-Aggregation-App

# 2. サーバーを起動
./start_server.sh

# 3. ブラウザでアクセス
# http://localhost:3000
```

> [!TIP]
> コードを変更した場合は `./start_server.sh restart` で再ビルド＆再起動できます。

## 画面の説明

本アプリは3つのメイン画面で構成されています。

### ホーム画面

タスクの記録と管理を行うメイン画面です。

| 機能 | 説明 |
|-----|------|
| **タスク追加** | 画面下部の入力欄にタスク名を入力して「追加」ボタンを押す |
| **計測開始** | タスク一覧からタスクをクリックすると計測が開始される |
| **計測停止** | 画面最下部の「計測ストップ」ボタンを押す |
| **タスク検索** | 検索ボックスでタスクを絞り込み |
| **並び替え** | 作成日時順・アクティビティ順・50音順の切り替え |
| **記録の編集** | タイムラインの記録にある「編集」ボタンから開始・終了時刻を修正 |
| **記録の削除** | タイムラインの記録にある「削除」ボタンで記録を削除 |

#### 基本的な使い方

1. タスク名（例: `メール対応`）を入力して追加
2. タスクをクリックして計測開始 → 自動で時間が記録される
3. 別のタスクをクリックすると、前のタスクを自動停止して切り替え
4. 全ての作業が終わったら「計測ストップ」で停止

> [!NOTE]
> タスクは一度追加すれば繰り返し使えます。不要になったタスクはアーカイブ（非表示）にできます。

### カテゴリー管理画面

タスクをカテゴリーにグループ化して、集計を整理する画面です。

| 機能 | 説明 |
|-----|------|
| **カテゴリー作成** | 画面上部の入力欄にカテゴリー名を入力して「追加する」 |
| **タスクの割当** | 右側の「未割当タスク」から「割当」ボタンでカテゴリーに紐付け |
| **合計時間表示** | 各カテゴリーの合計稼働時間が自動計算される |
| **カテゴリー削除** | カテゴリーを削除しても、紐づいたタスクは削除されず未割当に戻る |

#### 使い方の例

1. カテゴリー「社内会議」「開発作業」「ドキュメント作成」などを作成
2. 既存タスクを適切なカテゴリーに割り当て
3. レポート画面で、カテゴリー単位の集計を確認

---

### レポート画面

稼働時間の集計結果を確認する画面です。4つの表示モードがあります。

#### 日別レポート
- 日ごとの稼働時間の合計を表示
- ドーナツチャートでカテゴリー別の時間配分を可視化
- タイムテーブルで各記録の開始・終了時刻を確認
- CSV ダウンロード機能つき

#### 週別レポート
- 月曜始まりの週ごとに集計
- カテゴリー別・タスク別の内訳を表示
- 30分単位の丸め表示（15分未満→0分、15〜44分→30分、45分以上→繰り上げ）

#### 月別レポート
- 月ごとの集計を表示
- 週別レポートと同じフォーマット

#### カテゴリ別レポート
- カテゴリーごとに全期間の稼働時間を集計
- 各カテゴリーを展開すると週ごとの内訳を確認可能

> [!TIP]
> 各レポートカードのヘッダーをクリックすると詳細が展開されます。

## 多言語対応

画面右上の **EN / JA** ボタンで英語と日本語を切り替えできます。設定はブラウザに保存され、次回アクセス時も維持されます。

## よくある質問

### Q: データはどこに保存されますか？
**Docker の場合**: プロジェクトフォルダ内の `data/` ディレクトリにSQLiteファイルとして保存されます。コンテナを削除してもデータは保持されます。

**Electron の場合**: アプリのデータフォルダ内に自動保存されます。

### Q: データのバックアップ方法は？
`data/` フォルダ内の `.sqlite` ファイルをコピーしてください。

### Q: ポート番号を変更したい
`docker-compose.yml` の `ports` セクション、または環境変数 `PORT` を変更してください。

```yaml
# docker-compose.yml
ports:
  - "8080:3000"  # ホスト側を8080に変更
```

### Q: タイムゾーンがずれる
`docker-compose.yml` の `TZ` 環境変数を確認してください（デフォルト: `Asia/Tokyo`）。
